Setup

Everything you need to get StravaXEng running locally from scratch.

Prerequisites

  • Python 3.11+
  • PostgreSQL (local or remote)
  • A Strava API application — see Strava API

1 — Clone the repository

git clone <repo-url>
cd StravaXeng

2 — Create and activate the virtual environment

# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python -m venv .venv
source .venv/bin/activate

3 — Install dependencies

pip install -r requirements.txt
Packages installed: Django 4.2, requests, python-dotenv, psycopg2-binary.

4 — Configure environment variables

Edit .env in the project root. All variables are listed below:

# Strava
STRAVA_CLIENT_ID=<your client id>
STRAVA_CLIENT_SECRET=<your client secret>
STRAVA_REFRESH_TOKEN=<your refresh token>

# Django
SECRET_KEY=<long random string>
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

# PostgreSQL
DB_NAME=stravaxeng
DB_USER=postgres
DB_PASSWORD=<your password>
DB_HOST=localhost
DB_PORT=5432

Generate a SECRET_KEY:

python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

5 — Create the PostgreSQL database

-- run in psql
CREATE DATABASE stravaxeng;

6 — Run migrations

python manage.py migrate

7 — Create an admin superuser (optional)

python manage.py createsuperuser

8 — Sync activities from Strava

# one page = 30 activities
python manage.py sync_activities

# fetch more pages
python manage.py sync_activities --pages 5

See Management Commands for details.

9 — Start the development server

python manage.py runserver
